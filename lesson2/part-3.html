Part 3: Adding Styles with Classes and IDS

Adding a background color to a div is a beginning, but there are a myriad of rules an object can be given and an infinite way to apply rules to objects to produce a website.

Inline styles is one way, but adding a class or an ID is a better way

To add a class to an object you simply add the attribute of "class", like this:

<div class="some-class"> some stuff </div>

Classes are identifiers for CSS to hook into. It draws a connection between CSS and an object. It's like the hookshot in Legend of Zelda, but with code.

We can then add a style object to the page using the <style> tag.

<style>
</style>
<div class="some-class"> some stuff </div>

We can hook styles in a style object to objects on the page via the class. In order to drawn the connection, a period is used.

<style>
  .some-class{background:blue;}
</style>

<div class="some-class"> some stuff </div>
some stuff





We can also use the ID attribute to select an element on the page

<style>
  .some-class{background:blue;}
  #someID{background-color:red;}
</style>

<div id="someID" class="some-class"> some stuff </div>
some stuff

ID's can be used in tamdem with classes but notice about how the ID overrides the class. This is because IDs hold move weight than classes. This is because they are meant to be unique to their object. A good rule of thumb is that there can only be one ID per page.

When writing CSS in this manner, the hooks from CSS to HTML are called "selectors".
